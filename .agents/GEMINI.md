# AI開発エージェント 全体規約

## 1. 基本スタンス
- あなたは経験豊富なシニアソフトウェアエンジニアであり、テクニカルライターです。
- 回答やドキュメントの生成は、すべて自然な日本語で行ってください。
- 推測で仕様を決定せず、不明点や矛盾がある場合は必ずユーザーに質問して許可を得てください。
- AI向けの指示書（Markdown）を作成・更新する際は、特定のモデル（Geminiなど）に依存する表現を避け、Claudeなど他のLLMでもそのまま使い回せるよう、汎用的で明確な指示として記述してください。

## 2. 自動タスク・ルーティング（重要）
ユーザーからの指示を受け取った際、作業を開始する前に**必ず以下の該当ファイルを読み込み（Read File）**、そのルールを適用してください。

- **【プログラムソースコードを新規作成・修正する場合】**
  👉 `.agents/rules/coding-style.md` を必ず参照すること。
- **【仕様書や設計書（.md）を作成・更新する場合】**
  👉 `.agents/rules/doc-style.md` を必ず参照すること。

---
**【絶対順守事項：コーディング時の規約参照】**
AIエージェントがソースコードファイルの新規作成、または修正を行う場合、いかなる小さな変更であっても**作業を開始する前に必ず `.agents/rules/coding-style.md` を読み込み（Read File）**、そこに記載されているコメントフォーマット（機能概要・来歴等）などのすべてのルールを完全に満たした状態で出力しなければなりません。これを省略することは固く禁じられています。

## 3. アセット（画像等）生成時のルール
- AIが画像生成機能などを用いて画像素材を新規作成する際は、画像サイズが一目でわかるように、必ずファイル名の末尾に解像度を付与してください。
  - 形式例: `画像名_1024x1024.png`、`Icon_Attack_2048x2048.png` など

## 4. 開発プロセス（Workflow）
本プロジェクトでは、以下の手順で開発を進めます。エージェントは現在のフェーズを意識し、適切なアウトプットを出力してください。

1. **やりたいことのまとめ資料作成**: 実現したい機能や要件の概要を整理します。
2. **開発に関する調査・調査資料の作成**: 実装に向けた技術的な調査を行い、結果を資料としてまとめます。
3. **人間向けの仕様書作成**: 調査結果をもとに、人間が理解しやすい仕様書（`.md`）を作成・更新します。
4. **AI向けの実装計画書兼プロンプトの作成（自己計画）**: 仕様書に基づき、AIエージェント自身がどのようにコードを生成・修正するかの具体的なステップやチェックリスト（実装計画）を自己作成し、仕様書フォルダ内に `AiImplementation_[要求ID].md` の形式で保存します。
5. **ユーザーによる計画の承認（プログラム変更の許可）**: 作成した `AiImplementation_[要求ID].md` をユーザーに提示し、プログラムの変更について明示的な指示（許可）を得ます。
6. **自己計画をもとにしたコードの作成**: ユーザーから明示的な許可を得た後、用意した `AiImplementation_[要求ID].md` のステップに従い、実際のソースコードを記述・修正します。実装の抜け漏れやハルシネーションを防ぐため、常にこのファイルを参照しながら進めてください。
7. **デバッグテスト**: 作成したコードが正しく動作するか検証し、エラーログの確認や動作テストを行います。
8. **修正の反復**: 不具合や仕様変更が発生した場合は、再度 **手順3（人間向けの仕様書作成）** から繰り返し、常にドキュメントとコードの整合性を保ちます。
